# Code Review and Lint

Use this skill when reviewing or improving code quality.

## Rules

1. Prefer readability over cleverness.
2. Remove dead code and duplication.
3. Simplify overly complex conditionals and components.
4. Ensure naming is clear and consistent.
5. Enforce formatting and lint-friendly patterns.
6. Flag unsafe type usage and silent failure paths.
7. Improve error handling where weak.
8. Encourage reusable abstractions only when they reduce real duplication.
9. Avoid unnecessary dependencies.
10. Keep refactors safe and incremental.
11. Preserve behavior unless explicitly asked to change it.
12. Leave the codebase cleaner than before.

## Output expectations

- Code should become easier to understand.
- Refactors should reduce future maintenance cost.
- Output should be practical, not academic.
