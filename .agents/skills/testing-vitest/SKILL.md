# Vitest Testing

Use this skill for unit and integration-style tests in Vitest.

## Rules

1. Test behavior, not implementation details.
2. Prefer clear arrange-act-assert structure.
3. Keep tests small and focused.
4. Use descriptive test names.
5. Cover expected cases, edge cases, and failure cases where valuable.
6. Avoid brittle mocks unless necessary.
7. Prefer realistic input data.
8. Do not add meaningless snapshot tests.
9. Keep test setup minimal and reusable.
10. Favor confidence and readability over excessive test quantity.

## Output expectations

- Tests should be easy to read and maintain.
- Important business logic should be covered.
- Avoid noisy and low-value tests.
