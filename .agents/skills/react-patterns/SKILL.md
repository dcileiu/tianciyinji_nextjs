# React Patterns

Use this skill for React component design and refactoring.

## Rules

1. Build small, focused, reusable components.
2. Keep components single-purpose.
3. Prefer props-driven design over hardcoded values.
4. Avoid deeply nested JSX when possible.
5. Extract repeated UI blocks into subcomponents.
6. Keep local state minimal and derive values when possible.
7. Lift state up only when necessary.
8. Use descriptive prop names.
9. Avoid prop drilling when composition can solve it cleanly.
10. Prefer controlled forms for complex interactions.
11. Add accessibility basics such as labels, button types, aria attributes when needed, and keyboard-friendly interactions.
12. Avoid large monolithic components.

## Output expectations

- Components should be easy to read.
- Props interfaces should be clear.
- State and effects should be simple and justified.
- Avoid unnecessary re-renders and duplicated logic.
