# 开放媒体解析接口文档

适用范围：

- 公众号文章解析
- 抖音去水印解析
- 小红书去水印解析
- 哔哩哔哩视频解析（统一入口）
- 快手图文 / 视频解析（统一入口）

当前线上服务基址：

```text
https://wallpaper.api.itianci.cn
```

OpenAPI 规范地址：

```text
https://wallpaper.api.itianci.cn/api/v1/open/media/openapi.json
```

## 1. 接口概览

后端目前提供两种接入方式：

1. 统一入口，自动识别平台并返回统一结构
2. 平台专用接口，分别返回公众号 / 抖音 / 小红书的专用结构

推荐：

- 新项目接入，优先使用统一入口 `/api/v1/open/media/extract`
- 如果你已经按平台分别处理字段，使用专用接口更直接

## 2. 通用说明

### 2.1 请求格式

- 请求方法：`POST` 为主
- 请求头：`Content-Type: application/json`
- 编码：`UTF-8`

### 2.2 通用返回包裹

除统一入口外，公众号 / 抖音 / 小红书专用接口都使用相同的外层返回格式：

```json
{
  "code": 0,
  "data": {},
  "msg": ""
}
```

说明：

- `code = 0` 表示成功
- `code != 0` 表示失败
- 失败时通常 `data = null`，并在 `msg` 返回错误说明

### 2.3 限流

- 平台专用接口：同一 IP `5` 秒内限制 1 次
- 统一入口：匿名请求同一 IP `1` 小时内最多 `6` 次
- 统一入口：携带合法 API Key 时不占用匿名额度，也不受匿名限流限制

如果请求过于频繁，会返回：

```json
{
  "code": 429,
  "data": null,
  "msg": "Anonymous quota exceeded. Please retry later or use an API key."
}
```

统一入口会同时返回限流响应头：

| Header | 说明 |
| --- | --- |
| `X-RateLimit-Policy` | 当前访问策略，匿名请求形如 `anonymous; limit=6; window=3600`，API Key 请求形如 `api-key; limit=unlimited` |
| `X-RateLimit-Limit` | 匿名窗口内总额度，默认 `6` |
| `X-RateLimit-Remaining` | 当前匿名窗口剩余额度 |
| `X-RateLimit-Reset` | 当前匿名窗口重置时间，Unix 秒级时间戳 |
| `Retry-After` | 仅 429 时返回，表示多少秒后可重试 |

### 2.4 `index` 参数说明

多个接口都支持 `index` 参数，用于只取指定资源。

规则：

- 类型：`number[]`
- 从 `1` 开始计数，不是 `0`
- 只保留正整数

示例：

```json
{
  "url": "分享口令或链接",
  "index": [1, 3]
}
```

表示只返回第 `1` 个和第 `3` 个资源链接。

### 2.5 `cookie` 参数说明

部分平台内容可能需要 Cookie 才能拿到完整资源：

- 公众号：可传 `cookie`，不传时后端会尝试使用 `WECHAT_COOKIE`，再没有则降级为默认值
- 抖音：可传 `cookie`，不传时后端会尝试使用 `DOUYIN_COOKIE`
- 小红书：可传 `cookie`，不传则直接按无 Cookie 方式解析
- 快手：可传 `cookie`，不传则直接按无 Cookie 方式解析
- 哔哩哔哩：当前实现不依赖 Cookie，服务端直接调用 B 站公开接口解析

## 3. 推荐统一入口

## 3.1 自动识别媒体解析

```text
POST /api/v1/open/media/extract
GET  /api/v1/open/media/extract
```

完整地址：

```text
https://wallpaper.api.itianci.cn/api/v1/open/media/extract
```

### 3.1.1 适用场景

当你的项目只想“给一个分享链接或整段分享文案，然后自动识别平台并返回统一字段”，推荐直接用这个接口。

当前统一入口支持：公众号、抖音、小红书、哔哩哔哩、快手。

各平台资源返回约定：

- 快手：短链和重定向后的 `chenzhongtech.com` 落地页都按快手处理；图文资源返回在 `images`，视频资源返回在 `videos`
- 哔哩哔哩：支持 `bilibili.com`、`m.bilibili.com`、`b23.tv` 短链，以及「【标题-哔哩哔哩】 https://b23.tv/xxx」这类完整分享文案；视频返回在 `videos`，封面图返回在 `images`；优先返回可直接下载的 MP4 单文件（通常为 360P），若 MP4 不可用则降级为 DASH 分片（`source` 为 `bilibili-dash-video`，可能带 `audio_url`）

哔哩哔哩目前**仅通过统一入口**解析，暂无类似抖音 / 小红书的专用 `/api/v1/bilibili/detail` 接口。

### 3.1.2 请求参数

支持以下入参别名：

- `url`
- `input`
- `text`

POST 示例：

```json
{
  "input": "复制打开抖音看看 https://v.douyin.com/xxxxxx/",
  "cookie": "",
  "imageFormat": "jpeg",
  "videoPreference": "resolution",
  "index": [1],
  "waitMs": 3500
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `input` / `url` / `text` | `string` | 是 | 分享链接或整段分享文案 |
| `cookie` | `string` | 否 | 平台 Cookie |
| `imageFormat` | `string` | 否 | 仅小红书生效，可选：`auto` `png` `webp` `jpeg` `heic` `avif` |
| `videoPreference` | `string` | 否 | 视频优先策略，可选：`resolution` `bitrate` `size`；仅抖音、小红书生效 |
| `index` | `number[]` | 否 | 仅返回指定序号的资源，1 开始 |
| `waitMs` | `number` | 否 | 页面等待时间，仅统一入口使用，默认 `3500`，实际限制 `1000~15000` |

哔哩哔哩请求示例：

```json
{
  "input": "【带女大学生线下配电脑 到底会踩多少坑？-哔哩哔哩】 https://b23.tv/wgNr4JZ"
}
```

### 3.1.3 可选鉴权

统一入口支持匿名访问和 API Key 访问：

- 匿名访问：无需鉴权，但同一 IP 默认 `1` 小时最多 `6` 次
- API Key 访问：需要在后台管理系统的「API Key」页面创建，携带合法且未过期的 key 后不受匿名限流限制

请求头可二选一：

- `x-api-key: <key>`

或：

- `Authorization: Bearer <key>`

如果请求里携带了 API Key 但 key 不存在、已删除或已过期，会返回 `401 Invalid or expired API key.`；如果完全不带 API Key，则按匿名请求处理。

后台管理：

```text
/admin/open-api-keys
```

新增 Key 时可填写：

| 字段 | 说明 |
| --- | --- |
| 名称 | 便于识别调用方，可留空 |
| 有效天数 | 按天计算，留空表示永久有效 |

完整 Key 只在创建成功后展示一次，数据库中仅保存 hash 和前缀。

### 3.1.5 返回字段说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `url` | `string` | 解析后的真实页面链接 |
| `source_url` | `string` | 原始输入中提取到的链接 |
| `share_text` | `string` | 原始分享文案 |
| `fetched_at` | `string` | 服务端解析时间，ISO 字符串 |
| `platform` | `string` | 平台标识，如 `wechat` `douyin` `xiaohongshu` `bilibili` `kuaishou` |
| `platform_name` | `string` | 平台名称 |
| `title` | `string` | 标题 |
| `account_name` | `string` | 账号名 / 公众号名 |
| `author` | `string` | 作者 |
| `publish_time` | `string` | 发布时间 |
| `summary` | `string` | 摘要 / 描述 |
| `content_text` | `string` | 纯文本内容 |
| `content_html` | `string` | HTML 内容 |
| `cover_image` | `string` | 封面图 |
| `images` | `array` | 图片资源列表 |
| `videos` | `array` | 视频资源列表 |

`images[]` 字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `url` | `string` | 图片地址 |
| `alt` | `string` | 图片说明，可为空 |

`videos[]` 字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `url` | `string` | 视频地址 |
| `poster` | `string` | 封面图 |
| `source` | `string` | 来源标记 |
| `referer` | `string` | 下载时可能需要的来源页 |
| `quality` | `string \| number` | 清晰度信息，部分平台返回 |
| `codec` | `string` | 编码信息，部分平台返回 |
| `audio_url` | `string` | 音频地址，部分平台返回 |

`videos[]` 常见 `source` 取值：

| `source` | 平台 | 说明 |
| --- | --- | --- |
| `douyin-api` | 抖音 | 无水印视频 |
| `xhs-api` | 小红书 | 无水印视频 |
| `kuaishou-api` | 快手 | 无水印视频 |
| `bilibili-mp4` | 哔哩哔哩 | MP4 单文件，推荐直接下载 |
| `bilibili-dash-video` | 哔哩哔哩 | DASH 视频分片，需与 `audio_url` 合并后才是完整视频 |

哔哩哔哩 `quality` 常见取值（B 站清晰度编号）：

| `quality` | 含义 |
| --- | --- |
| `16` | 流畅 360P |
| `32` | 清晰 480P |
| `64` | 高清 720P |
| `80` | 高清 1080P |

## 4.4 哔哩哔哩视频解析

哔哩哔哩**没有单独的平台专用接口**，请使用统一入口：

```text
POST /api/v1/open/media/extract
GET  /api/v1/open/media/extract
```

### 4.4.1 支持的输入形式

- 完整分享文案，例如：`【标题-哔哩哔哩】 https://b23.tv/xxxxxx`
- `b23.tv` 短链
- `https://www.bilibili.com/video/BVxxxxxxxx/` 视频页链接
- `https://m.bilibili.com/video/BVxxxxxxxx` 移动版链接

服务端会自动跟随 `b23.tv` 跳转并提取 `BV` 号，再调用 B 站公开 API 获取播放地址。

### 4.4.2 解析策略

1. 优先请求 MP4 单文件（`source = bilibili-mp4`，通常为 360P）
2. 若 MP4 不可用，降级为 DASH 高清分片（`source = bilibili-dash-video`，并可能返回 `audio_url`）

### 4.4.3 返回说明

统一入口返回结构与第 3.1 节一致，`platform` 固定为 `bilibili`。

- 视频地址在 `videos[]`
- 封面图在 `images[]` 和 `cover_image`
- 下载视频时请将 `videos[].referer` 作为 `Referer` 请求头，否则 CDN 可能返回 403

## 5. 媒体资源代理（可选）

若客户端无法直接访问各平台 CDN（跨域、缺 Referer、小程序合法域名限制等），可使用媒体代理接口：

```text
GET /api/v1/open/media/asset?url=<编码后的资源地址>&referer=<编码后的来源页>
```

完整地址示例：

```text
https://wallpaper.api.itianci.cn/api/v1/open/media/asset?url=https%3A%2F%2Fupos-sz-estgcos.bilivideo.com%2F...&referer=https%3A%2F%2Fwww.bilibili.com%2Fvideo%2FBV1SAjR62E4p%2F
```

说明：

- 仅允许 `https` 资源地址（服务端会自动将 `http` 升级为 `https`）
- 哔哩哔哩视频（`bilivideo.com`、`hdslb.com`）未传 `referer` 时，服务端会默认补 `https://www.bilibili.com/`
- 适用于图片、视频直链的转发下载，不支持 m3u8 转码

## 6. 错误码说明

常见错误码：

| code | HTTP 状态码 | 说明 |
| --- | --- | --- |
| `0` | `200` | 成功 |
| `400` | `400` | 参数错误，例如缺少 `url` |
| `401` | `401` | 请求携带的 API Key 不存在、已删除或已过期 |
| `429` | `429` | 触发限流 |
| `500` | `500` | 服务端解析失败或服务暂不可用 |

```ts
async function parseBilibili(input: string) {
  const res = await fetch("https://wallpaper.api.itianci.cn/api/v1/open/media/extract", {
    method: "POST",
    headers: {
      "Content-Type": "application/json"
    },
    body: JSON.stringify({ input })
  });

  const json = await res.json();
  if (json.code !== 0) {
    throw new Error(json.msg || "bilibili parse failed");
  }

  const data = json.data;
  if (data.platform !== "bilibili") {
    throw new Error(`unexpected platform: ${data.platform}`);
  }

  return data;
}
```

## 7.4 通过代理下载 B 站视频

```ts
function buildMediaAssetProxyUrl(
  mediaUrl: string,
  referer: string,
  base = "https://wallpaper.api.itianci.cn"
) {
  const params = new URLSearchParams({ url: mediaUrl });
  if (referer) params.set("referer", referer);
  return `${base}/api/v1/open/media/asset?${params.toString()}`;
}

// 解析完成后
const video = data.videos[0];
const downloadUrl = buildMediaAssetProxyUrl(video.url, video.referer || data.url);
```

## 8. 接入建议

- 如果前端希望统一处理，优先接 `/api/v1/open/media/extract`
- 如果你需要保留公众号 / 抖音 / 小红书各自独立页面，接各自专用接口
- 哔哩哔哩、快手请直接使用统一入口
- 如果要只取第一个视频，传 `index: [1]`（对 `videos` / `images` 合并后的资源序号生效）
- 对抖音、小红书、快手受限内容，建议预留 Cookie 透传能力
- 统一入口适合“粘贴分享文案直接识别平台”的场景
- B 站视频下载请使用返回的 `referer`，或在小程序 / 浏览器受限环境下走 `/api/v1/open/media/asset` 代理
