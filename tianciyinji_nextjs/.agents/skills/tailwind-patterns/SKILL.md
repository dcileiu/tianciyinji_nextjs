# Tailwind Patterns

Use this skill whenever writing or refactoring Tailwind CSS.

## Rules

1. Keep utility classes organized logically: layout, spacing, typography, color, effects, state.
2. Avoid extremely long unreadable class strings when extracting a reusable component would be better.
3. Prefer design consistency across spacing, radius, shadows, and typography.
4. Mobile-first responsive design by default.
5. Use semantic layout patterns: container, grid, flex, gap, spacing.
6. Avoid arbitrary values unless necessary.
7. Prefer reusable UI primitives for repeated styles.
8. Keep hover, focus, active, and disabled states clear.
9. Ensure good visual hierarchy with spacing and typography.
10. Use Tailwind classes in a way that is easy for humans to scan.

## Output expectations

- UI should look modern, clean, and balanced.
- Responsive behavior should be considered.
- Do not generate cluttered or inconsistent styling.
- Prefer elegant simplicity over flashy complexity.
