# shadcn/ui Patterns

Use this skill whenever working with shadcn/ui components.

## Rules

1. Prefer shadcn/ui primitives before building custom replacements.
2. Compose components cleanly instead of overcustomizing them.
3. Keep forms, dialogs, dropdowns, tables, and cards visually consistent.
4. Use the Button, Input, Select, Dialog, Sheet, Card, Badge, Table, Tabs, and Form primitives whenever appropriate.
5. Preserve accessibility and keyboard interaction.
6. Avoid unnecessary wrappers around shadcn/ui components.
7. Keep spacing and sizing consistent across components.
8. Prefer a polished SaaS-like visual style.
9. Use icons only where they improve clarity.
10. Do not create UI that looks overly dense or overly decorative.

## Output expectations

- UI should feel modern and professional.
- Components should be consistent and easy to maintain.
- Use shadcn/ui in a way that remains easy to extend later.
